- Use DIRECT top-level keys: `name:`, `states:`, `transitions:`, `variables:`, `startstate:`

  ```yaml
  states: [...]
  transitions:

  variables: [...]

  ```

**YAML SYNTAX REQUIREMENTS:**
- ONLY use approved template variables: {{userGoal}}, {{nodePoolContext}}, {{dataHandlingContext}}, {{agentTeam}}, {{workflowContext}}

- NEVER add extra quotes around template variables (e.g., NOT "{{userGoal}}" or ""{{userGoal}}"")
